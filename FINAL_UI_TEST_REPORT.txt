## 1. [HIGH SEVERITY] Flask Template Caching

**Flow:** Template Updates Not Reflected  
**Repro Steps:**  
1. Modify templates/admin/dashboard.html (e.g., change subtitle text)
2. Reload /admin page in browser
3. Old template content still displays

**Actual Behavior:**  
- Template file shows: "Единая modern-панель: core-метрики, ops-контроль и встроенный Graph Lab."
- Server renders: "Восстановленный UI по recovery-плану: метрики + интерактивные граф-режимы."
- Server is serving cached/stale template version
- Template modified at 22:00:07 but changes not reflected

**Expected Behavior:**  
- Template changes should be reflected immediately (in development)
- Or server should be restarted to load new templates (in production)

**Suggested Fix:**  
Option A (Development): Add to admin_app.py near Flask app initialization:
```python
app.config['TEMPLATES_AUTO_RELOAD'] = True
```

Option B (Production): Restart Flask server to load updated templates:
```bash
# Find Flask process
ps aux | grep admin_app.py

# Kill and restart
sudo systemctl restart nopolicybot-admin  # if using systemd
# OR
pkill -f admin_app.py && python3 admin_app.py
```

**Impact:**  
- Users see outdated UI
- New features in templates not visible
- Developers confused by discrepancy

**Priority:** HIGH - Affects all UI updates
